# Frontend: сторінка "Вхід" (route /auth/login)

Це вже саме твій репозиторій (`Team-project-frontend-main`), я не переписував
структуру — тільки заповнив порожні файли, що стосуються мого завдання, і
поправив кілька речей, які заважали всьому проєкту збиратись.

## Виправлення після скріншотів з Figma Dev Mode (Inputs/Buttons/Links)

Три речі, які я спочатку вгадав неправильно:

1. **Шрифт** — насправді всюди **Montserrat** (я спочатку взяв Nunito +
   Mulish навмання). Поправив у `src/app/layout.tsx`.
2. **Заокруглення кнопки** — в макеті кнопки з невеликим радіусом (~8px), а
   не "пігулка", як я зробив спочатку. Поправив `--radius-button` в
   `globals.css` і використання в `loginPage.module.css`.
3. **Колір** — семпл кольору "Color/Mantis Darker" з макета дав `#254c24`
   (я спочатку взяв дещо інший відтінок зеленого, `#1f3a2e`). Поправив
   `--color-primary`/`--color-primary-dark`.

Якщо в команди є точні значення дизайн-токенів (наприклад, expo із Figma
Variables/Styles) — краще звірити з ними, я орієнтувався на семпл пікселів зі
скріншотів Dev Mode.

## Виправлення після твого скріншоту з Figma Dev Mode (макет картки)

Спочатку я домалював двоколонковий макет із зображенням зліва на десктопі —
його в реальному дизайні не було. За твоїм скріншотом з Dev Mode переробив
на те, що там справді намальовано: **центрована картка з рамкою**
(`.card` в `authLayout.module.css`), без split-screen, а
"© 2025 Природні Мандри" тепер всередині картки, під лінією, а не окремим
компонентом.

## Мій пункт (п.2 ТЗ, фронтенд)

- `src/app/auth/login/page.tsx` — сама сторінка входу: Formik + Yup
  (валідація email/пароль за правилами з ТЗ), запит через
  `@tanstack/react-query` (`useMutation`) на `POST /auth/login` (беру шлях з
  вже готового `src/lib/api/endpoints.ts` — там і так все правильно
  налаштовано, я нічого не міняв), помилки — через `react-hot-toast`, показ/
  приховування пароля, індикатор завантаження на кнопці, після успіху —
  `queryClient.invalidateQueries(['currentUser'])` + редірект на "/".
- `src/app/auth/login/loginPage.module.css` — стилі під макет (Formik-поля,
  кнопка-пігулка, стани помилки/фокусу).

## Спільне, що довелось доробити (використовується і на Реєстрації теж)

## ⚠️ Три реальні баги в репо, які я поправив (не пов'язані напряму з моїм
пунктом, але без них ніхто в команді не зміг би зібрати проєкт)

1. **`package.json` не мав `next`, `react`, `react-dom` і взагалі жодних
   TypeScript-пакетів** (`typescript`, `@types/*`) — тобто `npm install &&
   npm run build` не спрацював би в жодного учасника команди. Додав усе
   потрібне (версії — Next 15 / React 19, як передбачає `next.config.ts` і
   наявний `tsconfig.json`) і перегенерував `package-lock.json`.
2. **`src/app/components/icon/icon.tsx` ламав увесь білд.** Next.js App
   Router резервує назву файлу `icon.tsx` під автогенерацію фавікону
   (Metadata File Conventions), тож він намагався зібрати цей компонент як
   `route.js` для іконки сайту — і падав з помилкою "No response is returned
   from route handler". Перейменував файл на `svgIcon.tsx` (сам компонент і
   його експорт `Icon` не чіпав, тільки шлях імпорту).
3. **`eslint.config.mjs` імпортував `eslint-config-next/core-web-vitals` і
   `.../typescript` без розширення `.js`** — з встановленою версією
   `eslint-config-next` (15.5.20) ESM-резолвер не міг знайти файл. Додав
   `.js`. Лишилась одна дрібна нефатальна помилка ESLint
   (`nextVitals is not iterable`, схоже на несумісність версії конфіга з
   форматом `eslint.config.mjs`) — вона НЕ ламає `next build` (він
   завершується успішно, exit code 0), але саму лінтинг-перевірку варто
   комусь ще подивитись окремо.
